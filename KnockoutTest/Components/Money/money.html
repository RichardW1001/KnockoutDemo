Demo of a "money" binding that renders input as a currency amount.

There is validation in place enforcing a value between 0 and 50.

Note that there is a bug in the requirements; specifying a minimum greater than 0 is incompatible with the requirement to default to 0 if an empty value is provided and the requirement to revert to last value as the default value will fail validation and get into a logical loop! I have not attempted to deal with this edge case.
